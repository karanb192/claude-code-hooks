A reference catalog  ·  for Claude Code

Hooks for the agent loop, typeset & tested.

Five small programs that watch what Claude Code is about to do — and quietly intervene. Block a destructive shell command. Refuse to read a secret. Stage a file. Ping you on Slack. Each one a few hundred lines, exhaustively tested, ready to copy.

▾ Preface
Hooks are event-driven triggers that intercept Claude Code at specific points: before it writes a file, after it runs a command, when it needs your input. They accept JSON via standard input, return JSON via standard output, and can be written in any language a UNIX shell will execute. This catalog collects five tested implementations, a procedure for fitting them, and the reasoning behind each choice. It is meant to be read once and consulted many times.

§ 01Pre-tool-use
matcher: Bash Node · ~80ms 3 safety levels

block-dangerous-commands

Refuses rm -rf ~, fork bombs, curl | sh, dd to a raw disk, force-push to main, and the long tail of catastrophes that look benign in a one-liner. Ships with three configurable safety levels — critical, high, strict — so you can choose where the line is.

// Claude proposes: tool: Bash command: rm -rf ~ ─── verdict ───→
⊘ blocked "deletes home directory"
§ 02Pre-tool-use
matcher: Read|Edit|Write|Bash Node · ~80ms

protect-secrets

Stops Claude from reading, modifying, or quietly exfiltrating .env files, SSH keys, AWS credentials, kubeconfig — anything whose presence on disk is a load-bearing assumption. Catches the obvious paths and the clever ones (a cat piped to curl is still exfiltration).

// Claude proposes: tool: Read path: .env.production ─── verdict ───→
⊘ blocked "sensitive credential file"
§ 03Post-tool-use
matcher: Edit|Write Node · ~60ms

auto-stage

After Claude edits or creates a file, runs git add on it. Removes one tiny accounting step from your loop and makes git diff --cached the canonical "what did the agent just do" view.

// after Edit succeeds: $ git add src/api.ts ─── result ───→
✓ staged // next: review with $ git diff --cached
§ 04Notification
matcher: permission_prompt | idle_prompt Node · ~80ms

notify-permission

Pings a Slack channel when Claude is sitting on a permission prompt or has gone idle waiting on you. Lets you start a long agentic run, switch tabs, and trust that you'll be recalled when needed.

// Claude is waiting on: event: permission_prompt tool: Bash (sudo) ─── slack ───→
↗ #claude-runs "Claude needs you in repo X"
§ 05Diagnostic
utility Python not for production

event-logger

Wires into every event and writes the JSON payloads to a log so you can see what Claude Code actually sends. Use it to discover the shape of an event before writing a real hook against it. The thing you reach for first.

[12:04:09] PreToolUse { tool_name: "Bash",   tool_input: { ... } } ─── then ───→
write hook with confidence.
§ III   Apparatus

How to fit a hook.

three steps
~ 60 seconds
no build step
Procedure

Copy the script. Register it. Restart.

Hooks are plain executables. Claude Code calls them, pipes JSON to stdin, and waits for an exit code. There is no framework, no SDK, no plugin manifest. The wire format is the contract.

i.

Place the script

Anywhere your shell can chmod +x. The conventional location is ~/.claude/hooks/.

$ mkdir -p ~/.claude/hooks
$ cp hook-scripts/pre-tool-use/block-dangerous-commands.js \
     ~/.claude/hooks/
ii.

Register it

In .claude/settings.json, bind the hook to the event and matcher you care about.

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "node ~/.claude/hooks/block-dangerous-commands.js"
      }]
    }]
  }
}
iii.

Restart Claude Code

The hook is now active. Try it: ask Claude to run something destructive and watch it refuse.

$ claude
› please rm -rf ~
… hook blocked: deletes home directory.
§ IV   Notes

Marginalia & tradeoffs.

N. 1  ·  Safety

Three levels of paranoia.

Each safety hook ships with a SAFETY_LEVEL constant. Pick one and forget it.

critical only catastrophes — rm -rf ~, fork bombs, dd to disk
high + risky — force push main, secrets exposure, hard reset
strict + cautionary — any force push, sudo rm, docker prune
N. 2  ·  Runtime

Pick the language for the event rate.

Hooks fire often. Cold-start time becomes latency.

Bashsimple guards, tiny logic10–20 ms
Nodeper-tool checks, JSON-heavy50–100 ms
Pythonsession events, infrequent200–400 ms
N. 3  ·  Exits

Two numbers do the work.

The wire protocol is forgiving. The exit code is everything.

0pass — let the tool run
2block — return the message to Claude as guidance
·other — non-blocking warning, logged
